## Why

現在のゲームは「足し算の答え（和）を入力する」通常モードのみを提供しています。学習効果を高めるため、足し算の逆算思考を鍛える「虫食い算モード」を追加します。

虫食い算モードでは、足し算の式の一部（被加数または加数）が `□` で隠され、プレイヤーは隠された数を答えます（例: `5 + □ = 8` → 答えは `3`）。和から逆算する力を養うことで、繰り上がり・繰り下がりの理解や数感覚の定着を促します。

既存のステージ制・スコア・ダメージ・必殺技・演出の仕組みはそのまま流用し、出題形式と回答判定のみをモードに応じて切り替えます。

## What Changes

- スタート画面に「通常モード」「虫食いモード」の2つの開始ボタンを設け、選んだモードでバトルを開始できるようにする（ニックネーム入力なしのゲスト即時開始は維持）。
- 選択したゲームモード（`normal` / `blank`）を、暗号化セッショントークン・キャリートークン・スコア保存まで一貫して引き継ぐ。
- 虫食いモードの問題生成: 既存のステージ難易度規則で生成した足し算 `a + b = c` に対し、被加数（1項目）または加数（2項目）のいずれかをランダムに `□` として隠す。隠した数（＝正解）はクライアントへ送信せず、答えの漏洩を防ぐ。
- 虫食いモードの表示: `□ + b = c` または `a + □ = c` の形式で出題し、`□` を視覚的に強調する。プレイヤーは隠された数をテンキー／キーボードで入力する。
- 回答判定をモード対応にする: 通常モードは「和」を、虫食いモードは「隠された数（1項目なら被加数、2項目なら加数）」を正解とする。サーバー側のセッショントークン再検証も、各問題のモードと `□` 位置に基づいて整合性を検証する。
- ランキングをモード別に分離する: スコア保存時にモードを記録し、ランキング取得・表示を `normal` / `blank` ごとに行う。
- スコア・ダメージ・タイムボーナス・鬼HP・必殺技ゲージの各計算式は両モードで共通とする（ゲージ加算量も足し算 `a + b` ベースで両モード同一）。

## Capabilities

### Modified Capabilities
- `addition-battle`: ゲームに「通常モード」「虫食いモード」の2モードを導入。問題生成・配信、回答判定とセッション検証、スタート画面、スコア保存、ランキング表示をモード対応に拡張する。

## Impact

- 影響を受ける capability: `addition-battle`
- 影響を受ける主なコード:
  - `src/pages/api/[...path].ts`（問題生成・公開ペイロード・検証・スコア保存・ランキング取得をモード対応に）
  - `src/scripts/game.js`（モード保持、虫食い表示用の問題データ処理、回答判定、登録/ランキング遷移へのモード受け渡し）
  - `src/pages/index.astro`（2ボタン化とモード指定での開始）
  - `src/pages/game.astro`（虫食い式の表示）
  - `src/pages/ranking.astro`（モード別ランキング表示）
  - `init.sql`（`scores` テーブルへ `mode` 列追加のマイグレーション）
- 既存の通常モードの挙動・スコアは後方互換を維持する（モード未指定時は `normal` とみなす。既存スコアは `normal` 扱い）。
